1. Présentation Générale de l'Application

Inventory Pro est une application web full-stack de gestion d'inventaire conçue pour permettre aux entreprises de suivre leurs produits, stocks, mouvements d'entrée/sortie, sites d'entreposage et alertes de stock en temps réel. L'application est développée dans le cadre d'un Projet de Fin d'Études (PFE).

Objectifs principaux

  • Gestion complète des produits (CRUD, catégorisation, code-barres, images)
  • Suivi des stocks par site avec seuils d'alerte configurables
  • Enregistrement et traçabilité des mouvements de stock (entrées/sorties)
  • Gestion multi-sites (entrepôts, magasins, centres de distribution)
  • Système d'alertes automatiques basé sur les seuils de stock
  • Scanner de codes-barres intégré (caméra ou QR code externe)
  • Gestion des utilisateurs avec contrôle d'accès basé sur les rôles (RBAC)
  • Suggestions de réapprovisionnement basées sur les seuils
  • Tableau de bord (Dashboard) avec statistiques en temps réel

Architecture Frontend

Pattern : Standalone Component Architecture

L'application Angular utilise des composants autonomes (standalone components) avec le lazy loading pour chaque module fonctionnel. L'état est géré par des Angular Signals (réactivité fine).

Architecture Backend

Pattern : Clean Architecture + CQRS

Le backend est structuré en 4 couches distinctes suivant le principe de Clean Architecture :

  • Domain — Modèles métier, interfaces, commandes, requêtes
  • Application — Contrôleurs API, handlers, DTOs, services, Startup
  • Data — DbContext (EF Core), Migrations, Repositories
  • Infra — Injection de dépendances, configuration infrastructure

7. Système d'Authentification & Autorisation

Authentification (JWT)

  • Le backend émet des tokens JWT signés avec HMAC-SHA256
  • Durée de validité : 1440 minutes (24 heures)
  • Issuer : InventaireApi | Audience : InventaireApp
  • Le token est stocké côté client et envoyé via le header Authorization: Bearer <token>

Autorisation (RBAC)

L'application implémente un système RBAC (Role-Based Access Control) avec 3 rôles prédéfinis :

RôleNom BackendCouleurDescription
Admin admin 🔴 Rouge (#f44336) Accès complet au système, gestion des utilisateurs et rôles
Gestionnaire de Stock gestionnaire_de_stock 🔵 Bleu (#2196f3) Gestion des mouvements, alertes, rapports. Pas d'accès à la gestion des utilisateurs
Opérateur operateur 🟢 Vert (#4caf50) Opérations de base : scan, consultation produits, entrée/sortie

Protection des Routes (Frontend)

Les guards Angular protègent les routes :

  • authGuard — Vérifie l'authentification
  • adminGuard — Restreint aux administrateurs
  • stockManagerGuard — Restreint aux gestionnaires de stock
  • permissionGuard('permission') — Vérifie une permission spécifique
